SECTION 1 · 프로젝트 이해

현재 사이트 현황 분석

🖥️

PC/모바일 이원화

PC 웹(PHP)과 모바일(C언어)의 개발 언어 이원화로 유지보수 효율성이 저하되고 있습니다. 동일한 기능을 두 코드베이스에서 각각 수정해야 하는 구조입니다.

📱

모바일 미최적화

별도 모바일 사이트 운영으로 관리 포인트가 2배로 증가하며 PC/모바일 간 UI·콘텐츠 일관성이 부족합니다. 사용자 경험 저하로 이어집니다.

🔒

보안 취약점

노후화된 시스템의 보안 취약점이 존재하며 최신 웹 표준을 준수하지 못하고 있습니다. SQL Injection, XSS 등 기본 보안 위협에 노출될 수 있습니다.

🎨

디자인 노후화

현재 디자인이 최신 트렌드에 뒤처져 브랜드 이미지가 저하되고 있습니다. 경쟁사 대비 시각적 완성도가 낮아 고객 신뢰도와 구매 전환율에 영향을 줍니다.

SECTION 1 · 프로젝트 이해

프로젝트 목적

1

반응형 웹 통합

하나의 소스코드로 PC·모바일·태블릿 모두 대응하여 관리 포인트를 단일화하고 운영 효율을 극대화합니다.

2

디자인 리뉴얼

3

보안 강화

설계 단계부터 보안 코딩을 적용하고 OWASP 기준을 준수하여 안전한 커머스 플랫폼을 구축합니다.

4

PHP 기반 유지보수 체계

내부 개발자가 익숙한 PHP 환경을 기반으로 구축하여 납품 후에도 자체 유지보수가 원활하게 이루어지도록 합니다.
